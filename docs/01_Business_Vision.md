# 01 业务需求与展望 (Business Vision)

本模块旨在阐明 MDDAP (常州园区数据报表平台) 的项目背景、核心目标及解决的业务痛点。

## 1. 项目背景 (Background)
随着常州园区业务的快速扩张，手工报表的局限性日益显现：数据滞后、口径不一、人工统计成本高。本项目旨在通过自动化、标准化的数据管道，建立园区的“数字大脑”。

## 2. 核心目标 (Core Objectives)
- **单一事实来源 (Single Source of Truth)**: 确保全园区共用一套 KPI 标准。
- **决策自动化**: 将繁琐的统计工作赋能于自动算法。
- **闭环追踪**: 深度连接数据与 Action（如通过 Planner 追踪 EHS 指标）。

## 3. 业务地图 (Business Map)
| 模块 | 核心目标 | 现状 |
| --- | --- | --- |
| **Delivery (交付)** | WIP 透明化、准时交付率 (OTD) 提升 | MES/SFC 数据割裂 |
| **Quality (质量)** | NC 实时监控、趋势预警 | 手工 Excel 汇总 |
| **Safety (安全/EHS)** | 零事故目标、隐患闭环 | 线下记录为主 |
| **Efficiency (效率)** | OEE 提升、人工效率对齐 | SAP 与现场数据未打通 |

## 4. 用户需求清单 (User Story)
- **生产经理**: 希望能实时看到产线 WIP 的积压情况并快速锁定逾期批次。
- **EHS 团队**: 希望隐患发现后，能自动触发到对应的责任人跟进。
- **质量工程师**: 希望一键分析 NC 的主要原因（Pareto）并查看历史趋势。
