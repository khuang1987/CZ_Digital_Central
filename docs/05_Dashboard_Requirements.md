# 05 报表页面需求 (Dashboard Specs)

本文档定义了前端各功能页面的视觉规范、交互逻辑及数据展示要求。

## 1. 全局设计规范 (UI/UX Standards)
- **配色标准**:
    - **Primary**: Slate 900 (背景), Blue 600 (行动项)。
    - **KPI**: Green 400 (正常), Red 500 (超期/异常)。
- **布局**: 统一采用网格 (Grid) 布局，顶部为 KPI Card 概览，主体为图表与明细。

## 2. WIP 交付看板需求
- **概览页 (Overview)**:
    - **Pareto Chart**: 展示前 30 个积压工序，支持点击联动。
    - **Status Card**: 总批量、总件数、逾期数、逾期且未开工数。
    - **Trend Chart**: 过去 15 天的 WIP 趋势。
- **明细页 (Details)**:
    - 提供全列展示（批次号、物料、工序、状态、停留时间等）。
    - 支持模糊搜索与多级过滤。

## 3. NC 质量看板需求
- **核心组件**:
    - **Top 5 原因统计**: 自动计算 Pareto 贡献度。
    - **Source Filter**: 支持切换 MES 或线下数据源。
    - **Dynamic Time Range**: 支持 MTD/YTD 快速切换。

## 4. 交互逻辑规范
- **联动响应**: 点击图表项应自动触发全局 Filter 的更新。
- **骨架屏 (Skeleton)**: 数据加载超过 500ms 时必须展示加载占位符。
- **异常提示**: API 超时或 SQL 错误需展示友好的错误提示，并支持“重试”。
