# 01 文档中心开发规范 (Docs Portal Standards)

本文档定义了 `CZ Digital Central` 文档中心及其 Markdown 渲染引擎的技术与视觉规范。

---

## 1. 布局与结构 (Portal Layout)

- **最大宽度**：预览区域限制在 `max-w-5xl` (1024px)，以确保长文阅读时的视线聚焦。
- **背景风格**：内容区使用 `bg-white` (Light) / `bg-slate-900` (Dark)，外围衬底使用 `bg-slate-50` / `bg-slate-950` 以产生悬浮层级感。
- **侧边导航**：
    - 自动递归扫描 `docs/` 目录。
    - 选中项背景：`bg-[#004b87]` (深蓝)，文字颜色：`text-white`。
    - 子目录文件：图标右上角带有蓝色圆点标记。

---

## 2. 交互式可折叠章节 (Collapsible Sections)

Markdown 渲染引擎将特定层级的标题转换为可折叠块：
- **H3 标题**：作为“模块级”折叠单元，左侧带有 `ChevronRight` 旋转箭头。
- **H4 标题**：作为“小节级”折叠单元。
- **悬浮控制**：右下角提供全局全开/全收按钮。

---

## 3. 层级缩进逻辑 (Indentation Guide)

为了体现内容的逻辑隶属关系，必须严格执行缩进：
- **H3 内容块**：向右缩进 `ml-6` (24px)。
- **H4 内容块**：向右缩进 `ml-12` (48px)。
- **样式要求**：所有段落、表格、卡片均需遵循此物理对齐规则。

---

## 4. 可视化集成 (Visualization)

- **Mermaid 渲染**：支持内嵌 `mermaid` 代码块，渲染为圆角背景容器。
- **技术卡片 (`> `)**：引用语法被渲染为带有蓝色侧边条的 Technical Card，用于标识元数据（如数据来源、版本等）。

---

## 5. 搜索索引逻辑 (Search Indexing)

- **递归深度**： `/api/docs/search` API 必须递归检索所有子文件夹。
- **高亮预览**：返回包含关键字的前后文片段 (Snippets)，支持点击直达。
