# 项目文件命名规范

## 1. 文件夹命名规则

### 1.1 格式
```
[序号]_[英文名]_[中文名]
```

### 1.2 规则说明
- 序号：两位数字，从01开始
- 英文名：使用首字母大写的英文名称
- 中文名：使用中文描述
- 各部分之间使用下划线"_"连接

### 1.3 示例
- `01_Define_定义阶段`
- `02_Measure_测量阶段`
- `03_Analyze_分析阶段`
- `04_Improve_改进阶段`
- `05_Control_控制阶段`
- `80_Presentation_项目展示`
- `90_Meeting_会议纪要`
- `99_Reference_参考信息`

## 2. 文件命名规则

### 2.1 格式
```
[序号]_[英文名]_[中文名].md
```

### 2.2 规则说明
- 序号：两位数字，从01开始
- 英文名：使用首字母大写的英文名称，多个单词直接连接
- 中文名：使用中文描述
- 文件扩展名：使用小写的.md
- 各部分之间使用下划线"_"连接

### 2.3 示例
- `01_ProjectDefinition_项目定义.md`
- `02_KpiSpecification_KPI详细说明.md`
- `01_DataCollectionPlan_数据收集计划.md`
- `01_AnalysisPlan_分析计划.md`
- `01_ImprovementPlan_改进方案.md`
- `01_ControlPlan_控制计划.md`

## 3. 命名注意事项

### 3.1 通用规则
- 使用小写字母
- 避免使用特殊字符
- 避免使用空格
- 保持命名的一致性
- 文件名应该清晰表达内容

### 3.2 缩写使用
- 仅使用广泛接受的缩写（如KPI）
- 避免使用不常见的缩写
- 缩写应保持大写

### 3.3 序号使用
- 文件夹序号：表示项目阶段或重要程度
- 文件序号：表示在同一文件夹中的顺序
- 序号使用两位数字（01-99）

## 4. 文件组织建议

### 4.1 文件夹结构
- 按项目阶段组织
- 使用编号前缀便于排序
- 保持层级结构清晰

### 4.2 文件组织
- 相关文件放在同一文件夹中
- 使用序号保持文件顺序
- 保持文件命名的一致性
