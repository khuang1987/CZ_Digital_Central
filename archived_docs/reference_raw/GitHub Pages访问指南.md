## 🌐 访问地址

### 标准访问地址

部署到 GitHub Pages 后，您的文档可以通过以下地址访问：

```
https://您的用户名.github.io/250418_MDDAP_project/
```

**示例：**
```
https://khuang1987.github.io/250418_MDDAP_project/
```

---

## 📋 访问步骤

### 步骤 1：确认部署成功

1. **检查 GitHub 仓库**
   - 打开您的 GitHub 仓库：`https://github.com/您的用户名/250418_MDDAP_project`
   - 确认存在 `gh-pages` 分支
   - 检查该分支中是否有 `index.html` 文件

2. **检查 GitHub Pages 设置**
   - 进入仓库 **Settings** → **Pages**
   - 确认 **Source** 设置为 `gh-pages` 分支
   - 确认 **Branch** 选择 `/ (root)`

---

### 步骤 2：获取访问地址

**方法 1：从 GitHub 设置获取**

1. 进入仓库 **Settings** → **Pages**
2. 在页面顶部会显示：
   ```
   Your site is live at https://您的用户名.github.io/250418_MDDAP_project/
   ```
3. 点击该链接即可访问

**方法 2：手动构建地址**

格式：
```
https://[您的GitHub用户名].github.io/[仓库名]/
```

示例：
```
https://khuang1987.github.io/250418_MDDAP_project/
```

## ⚠️ 常见问题

### 问题 1：显示 404 Not Found

**可能原因：**
1. 部署还未完成（需要等待 1-5 分钟）
2. GitHub Pages 未启用
3. `gh-pages` 分支不存在
4. 访问地址不正确

**解决方法：**

1. **等待几分钟后重试**
   ```
   部署后通常需要 1-5 分钟才能生效
   ```

2. **检查 GitHub Pages 设置**
   - Settings → Pages
   - Source 应选择 `gh-pages` 分支
   - 点击 Save

3. **检查分支是否存在**
   ```bash
   # 在 GitHub 仓库页面查看分支
   # 应该能看到 gh-pages 分支
   ```

4. **验证访问地址**
   - 确认用户名和仓库名正确
   - 确认 URL 格式正确

## 🔄 更新后访问

### 更新文档后

每次更新文档并重新部署后：

1. **等待 1-5 分钟**
   - GitHub Pages 需要时间更新
2. **清除浏览器缓存**
   - 按 `Ctrl + F5`（Windows）
   - 或 `Cmd + Shift + R`（Mac）
   - 强制刷新页面
3. **验证更新**
   - 检查内容是否已更新
   - 确认新功能是否生效

## 🔐 访问权限

### 公开访问

默认情况下，GitHub Pages 网站是**公开的**：
- ✅ 任何人都可以访问
- ✅ 不需要登录
- ✅ 不需要 GitHub 账号

### 私有仓库

如果仓库是私有的：
- ⚠️ GitHub Pages 仍然可以公开访问
- ⚠️ 但仓库内容本身是私有的
- 💡 建议：如果内容敏感，考虑使用其他部署方式
