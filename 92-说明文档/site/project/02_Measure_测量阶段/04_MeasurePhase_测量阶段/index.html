    测量阶段实施指南 / Measure Phase Implementation Guide¶

    前期信息收集 / Preliminary Information Collection¶

    需要确认的信息¶

    请在下方提供相关信息，以帮助我们制定更准确的测量阶段实施计划：

    1. 项目背景与目标
    2. 项目的具体目标是什么？ 对工厂的各类改善项目进行管理，统计当前的项目管理方法和管理软件，识别优劣，最终决定一套标准化的改善项目管理办法
    3. 项目涉及哪些系统？ 目前的范围是改善类型的项目，需要在第一次会议上明确是否需要包含其他的项目管理，比如新品开发项目，固资投资项目，或者其他可能的项目是否需要在这次的项目中包含

    4. 主要的项目干系人有哪些？ 工程师，各部门经理，技术人员，CI ，管理层等

    5. 现有系统情况

    6. 目前使用的主要系统有哪些？ 目前最基础的是excel 表，然后有人会使用ppt 的A3 模板做项目报告，planisware 用来统计和跟踪固资和改善项目回报等；project 主要用来做新品开发或者需要跟踪分解任务跟踪进展的项目
    7. 各系统之间的数据流转方式是什么？ 使用excel 统计汇总，手动填写或者上传

    8. 现有的数据收集方式和工具有哪些？ 手动统计，部分数据会用excel 保存在Sharepoint 使用powerBI 采集和展示

    9. 资源情况

    10. 可投入的人力资源有多少？ 5人左右的团队，来自CI 和生产部门，有最高领导支持，各部门经理配合
    11. 是否有专门的数据分析团队？ 不是专业的数据分析团队，有一定的数据分析能力，会使用powerBI powerapp 等工具

    12. 可用的工具和技术支持有哪些？ Office 套件，planner ，powerBI 等工具，

    13. 时间安排

    14. 测量阶段的总体时间要求是多久？ 1个月左右
    15. 是否有关键的时间节点需要考虑？ 暂时没有定

    16. 各部门的可参与时间情况如何？ 计划一周2小时左右，如果必要的情况可以延长

    17. 特殊要求

    18. 是否有特殊的数据安全要求？ 不能使用外部的服务器存放数据
    19. 是否需要考虑特定的合规要求？ 需要满足公司内部的要求
    20. 是否有其他需要特别注意的事项？

    请在以上问题下方填写您的回复，我们将根据您提供的信息调整和完善测量阶段的实施计划。

    一、数据收集计划 / Data Collection Plan¶

    1. 系统使用现状调查¶

    调查项目 收集方法 数据来源 频率
    各系统活跃项目数量 系统导出 Project/Planisware/Teams/Excel 每周

    系统功能使用情况 系统日志分析 各系统后台数据 每月
    A3模板使用情况 文件统计 SharePoint/本地存储 每周
    数据重复录入情况 工作观察 项目管理人员 每周

    2. 数据一致性评估¶

    评估项目 收集方法 样本量 周期
    项目状态一致性 跨系统数据比对 30个活跃项目 每周
    进度数据准确性 实地核实vs系统记录 15个关键项目 每周
    资源分配准确度 人力记录vs系统数据 所有项目组 每月
    项目收益计算一致性 跨系统数据比对 所有完成项目 每月

    跨系统数据同步时间 操作记录 系统组合/数据量 每次操作

    二、关键指标定义 / Key Metrics Definition¶

    1. 效率指标¶

    • 项目数据更新及时率 = 及时更新项目数/总项目数 × 100%
    • 数据整合工时 = Σ(数据收集+整理+校验时间)
    • 跨系统数据同步延迟时间 = 系统间数据不一致持续时间

    2. 质量指标¶

    • 数据一致性比率 = 数据一致项目数/总项目数 × 100%
    • 错误发生率 = 数据错误事件数/总数据录入次数 × 100%
    • 返工率 = 需重复录入次数/总录入次数 × 100%

    3. 满意度指标¶

    • 用户满意度评分（1-5分制）
    • 系统使用便利性评分
    • 报告获取便捷度评分

    五、注意事项 / Key Considerations¶

    1. 确保数据收集过程不影响正常工作
    2. 注意数据的保密性和安全性
    3. 保持与相关部门的沟通
    4. 及时记录异常情况
    5. 定期检查数据质量
    6. 重点关注跨系统数据同步问题
    7. 记录所有重复性工作
    8. 评估现有工具的使用效率
    9. 收集用户对现有系统的反馈
    10. 识别数据整合的瓶颈

    一、Process Thinking（问题-方案-进展）¶

    序号 发现的问题 方案与进展 预期收益 风险/依赖 Owner/截止

    2 字典/主数据不清晰（工序名、设备、产品号、工艺路线等），口径不一致 已完成14类工序名称标准化与映射；建立产品、工序、设备维表与唯一键规则；新增异常字典收敛流程（每周审查） 提升数据一致性，减少报表口径争议 历史数据遗留映射需补录 工艺&计划 / 持续
    3 无系统化的数据一致性与完整性验证，难以形成可信基线 建立校验规则：必填字段、时间逻辑、SFC⇄MES匹配率、标准时间覆盖率；实施周度质量报告（OnTime/Overdue判定、缺失项清单） 报表可信度提升；为改进阶段提供可用基线 需跨部门配合补充缺失字段 数据治理组 / 本月内

    补充说明： - 已完成生产记录模型与初版可视化；支持工序/设备/产品多维度SA分析。 - 具备按命名规则自动抓取“最新版本”能力，避免人工换源。 - 拟采用增量/分区刷新以支撑每周几十万行数据更新需求。

    二、测量目标与范围¶

    • 目标：形成“可复用、可审计、可度量”的数据资产与SA/效率类KPI的可信基线。
    • 范围：SFC批次记录、MES生产记录、标准时间/工艺路由、必要的主数据（产品、工序、设备、资源、区域）。

    三、数据流与系统边界¶

    1. 数据采集：SharePoint/OneDrive 中 Excel 最新文件 → Power Query 脚本（e1/e2/e3）。
    2. 数据处理：字段标准化（工序号补零、时间统一为DateTime、数量校验）；工序名称映射与主数据关联。
    3. 指标计算：按 10-SA指标/SA指标计算方法与定义.md 定义生成 LT/PT/PPT/DueTime/CompletionStatus 与 SA 达成率。
    4. 存储与消费：Power BI Dataset（事实表+维表）；报表/导出/API供后续改进阶段使用。

    四、关键指标口径（与文档对齐）¶

    • SA达成率、按期/逾期批次数、平均Lead Time、平均Process Time。
    • 更详细的KPI口径与数据来源，参见：
    • 02_Measure_测量阶段/02_KpiSpecification_KPI详细说明.md
    • 10-SA指标/SA指标计算方法与定义.md

    五、数据质量与一致性验证（执行中）¶

    • 必填字段校验：BatchNumber、CFN、Operation、TrackOutTime。
    • 时间逻辑校验：TrackOutTime ≥ StartTime ≥ EnterStepTime；0010工序LT起点使用 Checkin_SFC。
    • 关联性校验：SFC⇄MES匹配率、标准时间覆盖率、主数据映射命中率。
    • 异常清单：缺失/异常时间值、负数与极端数量、无法映射的工序/设备。
