## 🚀 使用方法（三选一）

### 方法 1：本地预览（推荐开发时使用）

**优点：** 修改后自动刷新，适合编辑和预览

```bash
# 1. 安装依赖（仅首次）
双击运行: 安装依赖.bat

# 2. 启动本地服务器
双击运行: 启动文档服务.bat

# 3. 浏览器自动打开 http://127.0.0.1:8000/
```

---

### 方法 2：生成静态网站（推荐分享使用）

**优点：** 生成 HTML 文件，可离线使用，可部署到服务器

```bash
# 1. 生成静态网站
双击运行: 生成静态网站.bat

# 2. 打开生成的文件
site/index.html （双击即可在浏览器中打开）

# 3. 整个 site 文件夹可以：
   - 复制到其他电脑离线使用
   - 上传到 Web 服务器
   - 分享给同事
```

---

### 方法 3：部署到服务器（推荐生产环境）

**优点：** 团队共享，随时访问

```bash

mkdocs build

# 2. 将 site 文件夹部署到：
   - 公司内网 Web 服务器
   - SharePoint
   - GitHub Pages
   - 其他 Web 服务器
```

## 🆘 常见问题

### Q: 安装依赖时提示找不到 Python？

**A:** 请先安装 Python 3.8+
- 下载地址：https://www.python.org/downloads/
- 安装时勾选 "Add Python to PATH"

---

### Q: 本地服务器无法启动？

**A:** 检查：
1. 是否已安装依赖？运行 `安装依赖.bat`
2. 端口 8000 是否被占用？关闭其他服务

---

### Q: 如何分享给同事？

**A:** 三种方式：
1. **生成 HTML**：运行 `生成静态网站.bat`，将 `site` 文件夹打包分享
2. **上传 SharePoint**：将 `site` 文件夹上传到 SharePoint
3. **部署服务器**：联系 IT 部署到内网服务器

---

### Q: 如何添加新页面？

**A:**
1. 在 `docs/` 对应文件夹下创建 `.md` 文件
2. 在 `mkdocs.yml` 的 `nav:` 部分添加链接
3. 重新生成静态网站
