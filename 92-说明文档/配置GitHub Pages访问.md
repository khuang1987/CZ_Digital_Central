## 📋 配置步骤

### 步骤 1：启用 GitHub Pages

1. **进入仓库设置**
   - 打开您的 GitHub 仓库：`https://github.com/khuang1987/250418_MDDAP_project`
   - 点击顶部导航栏的 **Settings**（设置）

2. **找到 Pages 设置**
   - 在左侧菜单中，向下滚动找到 **Pages**
   - 点击进入

3. **配置 Source**
   - 在 **Source** 部分：
     - 选择 **Branch: gh-pages**
     - 选择 **Folder: / (root)**
   - 点击 **Save**（保存）

4. **等待生效**
   - 通常需要 **1-5 分钟**
   - 页面顶部会显示：
     ```
     Your site is live at https://khuang1987.github.io/250418_MDDAP_project/
     ```

## ⚠️ 重要：私有仓库的访问限制

### 当前情况

从截图看到您的仓库是 **Private（私有）**，这会影响访问：

| 仓库类型 | GitHub Pages 访问 | 说明 |
|----------|------------------|------|
| **Public（公开）** | ✅ 任何人都可以访问 | 免费 |
| **Private（私有）** | ⚠️ 需要 GitHub 账号登录 | 需要付费计划 |

### 解决方案

#### 方案 1：使用 GitHub Enterprise（如果公司有）

如果您的公司使用 GitHub Enterprise：
- ✅ 私有仓库的 Pages 可以内部访问
- ✅ 需要公司 GitHub 账号登录
- ✅ 适合内部文档

#### 方案 2：改为公开仓库（推荐用于文档）

**如果文档内容不敏感，可以改为公开：**

1. **进入仓库设置**
   - Settings → General
   - 向下滚动到 **Danger Zone**

2. **更改可见性**
   - 点击 **Change visibility**
   - 选择 **Make public**
   - 确认更改

3. **结果**
   - ✅ GitHub Pages 可以公开访问
   - ✅ 不需要登录
   - ✅ 任何人都可以访问

**注意：** 改为公开后，仓库代码也会公开。如果只希望文档公开，可以考虑：
- 创建单独的公开仓库专门用于文档
- 或使用其他部署方式

---

#### 方案 3：使用其他部署方式

如果不想公开仓库，可以考虑：

1. **Netlify**（推荐）
   - 免费
   - 支持私有仓库
   - 自动部署
   - 访问地址：`https://您的项目名.netlify.app`

2. **Vercel**
   - 免费
   - 支持私有仓库
   - 快速部署

3. **Azure Static Web Apps**
   - 需要 Azure 订阅
   - 支持 Azure AD 认证
   - 适合企业环境

## 🆘 如果无法访问

### 问题 1：显示 404

**解决方法：**
1. 等待 1-5 分钟（首次部署需要时间）
2. 检查 Settings → Pages 配置是否正确
3. 确认 `gh-pages` 分支存在

### 问题 2：需要登录但无法访问

**解决方法：**
1. 确认您有仓库访问权限
2. 联系仓库管理员添加权限
3. 或考虑改为公开仓库

### 问题 3：页面显示但样式错乱

**解决方法：**
1. 检查 `mkdocs.yml` 中的 `site_url` 配置
2. 重新部署：`mkdocs gh-deploy --force`
