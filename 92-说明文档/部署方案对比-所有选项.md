## 🥇 方案 1：本地共享文件夹（最快）

### ⚡ 5分钟部署

**适用场景：**
- ✅ 快速测试
- ✅ 临时使用
- ✅ 团队都在同一网络
- ✅ 需要立即可用

### 操作步骤

**步骤 1：准备文件**
```bash
# 确保 site 文件夹已生成
cd 92-说明文档
# site 文件夹包含所有静态文件
```

**步骤 2：共享文件夹**

**方法 A：放在网络共享驱动器**
```
1. 将 site 文件夹复制到网络共享位置，例如：
   \\server\shared\数字化平台说明书\

2. 分享路径给团队：
   \\server\shared\数字化平台说明书\index.html

3. 团队成员：
   - 双击 index.html
   - 在浏览器中打开
```

**方法 B：使用本地共享**
```
1. 在您的电脑上创建共享文件夹
2. 右键 site 文件夹 → 属性 → 共享
3. 添加团队成员权限
4. 分享网络路径
```

**访问方式：**
```
\\您的电脑名\shared\site\index.html
```

### 优点 ✅
- 最快速（5分钟）
- 无需任何配置
- 完全本地控制
- 不需要 IT 支持

### 缺点 ❌
- 需要在同一网络
- 不支持远程访问
- 依赖您的电脑在线
- URL 不友好

## 🥇 方案 2：OneDrive 共享（推荐小团队）

### ⚡ 5分钟部署

**适用场景：**
- ✅ 小团队（<10人）
- ✅ 需要远程访问
- ✅ 已有 OneDrive
- ✅ 快速简单

### 操作步骤

**步骤 1：上传到 OneDrive**
```
1. 打开 OneDrive（您已经在使用）
2. 创建文件夹：数字化平台说明书
3. 将 site 文件夹的所有内容上传
```

**步骤 2：获取共享链接**
```
1. 右键 index.html
2. 选择"共享"
3. 设置权限：
   - 组织中的人员可以查看
   - 或添加特定人员
4. 复制链接
```

**步骤 3：创建快捷方式（可选）**
```
1. 在 OneDrive 中创建一个 .html 文件：
   跳转页面.html

2. 内容：
   <html>
   <head>
     <meta http-equiv="refresh" content="0; url=./site/index.html">
   </head>
   <body>正在跳转...</body>
   </html>

3. 分享这个跳转页面的链接
```

### 优点 ✅
- 快速简单
- 支持远程访问
- 已有的工具
- 自动同步

### 缺点 ❌
- OneDrive 查看器可能有限制
- URL 较长不友好
- 可能遇到预览问题

## 🏆 方案 3：Teams 共享（强烈推荐！）

### ⚡ 5分钟部署

**适用场景：**
- ✅ **最推荐方案！**
- ✅ 团队使用 Teams
- ✅ 需要协作讨论
- ✅ 简单易用

### 操作步骤

**步骤 1：在 Teams 中上传**
```
1. 打开 Teams 频道（例如：CZ Ops 团队）
2. 切换到"文件"标签
3. 上传 site 文件夹的所有内容
```

**步骤 2：添加为标签页**
```
1. 在频道顶部点击 "+"
2. 选择"网站"或"SharePoint"
3. 输入 index.html 的链接
4. 命名：数字化数据平台说明书
5. 保存
```

**或者：直接分享链接**
```
1. 右键 index.html → 复制链接
2. 在 Teams 频道发送链接
3. 团队成员点击即可访问
```

### 优点 ✅
- ⭐ 最方便团队使用
- ✅ 集成在日常工作流
- ✅ 支持讨论和反馈
- ✅ 移动端友好
- ✅ 不需要 IT 配置

### 缺点 ❌
- 仍然依赖 SharePoint 后台
- 可能遇到类似的预览问题

### 🔧 如果 Teams 也遇到下载问题

使用"在 Teams 中运行本地服务器"方式（见方案 8）

## 🚀 方案 5：内网文件服务器

### ⏱️ 1-2天部署（需要 IT 支持）

**适用场景：**
- ✅ 公司有内网 Web 服务器
- ✅ IT 部门支持
- ✅ 需要稳定的 URL
- ✅ 长期正式使用

### 操作步骤

**联系 IT 部门：**
```
主题：请求在内网 Web 服务器部署数字化平台说明书

IT 团队您好，

我们需要在内网 Web 服务器上部署一个静态网站。

需求：
• 静态 HTML/CSS/JS 网站
• 文件大小：约 XX MB
• 访问权限：CZ Ops 团队
• 期望 URL：http://intranet/data-platform 或类似

我们已准备好所有文件（site 文件夹）。

请告知：
1. 服务器地址和部署方式
2. 需要的文件格式（ZIP 或直接上传）
3. 预计上线时间

谢谢！
```

### 优点 ✅
- URL 简洁友好
- 稳定可靠
- 完全控制
- 性能好

### 缺点 ❌
- 需要 IT 支持
- 部署时间较长
- 可能需要申请流程

## ☁️ 方案 6：Azure Static Web Apps（推荐正式使用）

### ⏱️ 半天部署

**适用场景：**
- ✅ 长期正式使用
- ✅ 需要自定义域名
- ✅ 最佳性能
- ✅ 自动化部署

### 操作步骤

**步骤 1：准备 Azure 账号**
```
需要：
• Azure 订阅（可能需要 IT 申请）
• 或使用免费层级（有限制）
```

**步骤 2：部署到 Azure**
```bash
# 安装 Azure CLI
# 或使用 Azure Portal 网页界面

# 部署命令（简化版）
az staticwebapp create \
  --name czops-data-platform \
  --resource-group czops \
  --source ./site \
  --location eastasia
```

**步骤 3：配置**
```
1. 设置自定义域名（可选）
2. 配置 Azure AD 认证（推荐）
3. 设置访问权限
```

### 优点 ✅
- ⭐ 最佳性能和稳定性
- ⭐ 自动 HTTPS
- ✅ 可设置自定义域名
- ✅ Azure AD 集成
- ✅ 全球 CDN
- ✅ 自动备份

### 缺点 ❌
- 需要 Azure 订阅
- 可能需要 IT 审批
- 配置较复杂
- 有一定成本（通常很低）

### 成本
```
免费层级：
• 100 GB 带宽/月
• 通常足够内部使用

标准层级：
• 约 $9/月
• 无限带宽
```

## 📋 决策流程

### 问自己这些问题：

**Q1: 多急？**
- 今天就要 → Teams 或 OneDrive
- 本周内 → 加上 Azure 申请
- 不急 → 等 SharePoint IT 响应

**Q2: 多少人使用？**
- <10人 → OneDrive 或 Teams
- 10-50人 → Teams 或内网服务器
- >50人 → Azure 或正式内网服务器

**Q3: 使用多久？**
- 临时（几天） → 本地服务器或共享
- 中期（几个月） → Teams 或 SharePoint
- 长期（1年+） → Azure 或内网服务器

**Q4: 预算？**
- 零预算 → Teams/SharePoint/内网服务器
- 低预算 → Azure（约$9/月）
- 灵活预算 → Azure + CDN + 自定义域名
